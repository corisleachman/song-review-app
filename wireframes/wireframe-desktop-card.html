Desktop card view — key implementation rules
  • Card structure: flex-column. Artwork square on top (.cardThumb), opaque strip below (.cardBody). No overlay on the strip.
  • Tap artwork = play/pause. Tap anywhere else on card = navigate to song page.
  • Info overlay covers artwork area only — title strip and icons always accessible below.
  • Overlay: position:absolute inside .cardThumb, aspect-ratio:1/1 to match thumb height. Hidden via opacity:0; pointer-events:none. JS adds .cardInfoOverlayVisible.
  • Attention state: box-shadow: 0 0 0 2px #ff1493 — ring border, no layout shift.
  • Action pill: in .card-status-row alongside status pill, always visible in strip.
  • Icons: 4 buttons in a static horizontal row (position:static, always visible). Order: ℹ, rename, image, delete.
  • Playing state: EQ bars top-left of artwork, pause icon on play button, play button always visible (not hover-only).
  • Clicking ℹ again closes the overlay (toggle). Clicking outside the overlay also closes it.
