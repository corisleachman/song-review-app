                         QUICK START (20 MIN)
================================================================================

1. SETUP (5 min)
   npm install
   cp .env.local.example .env.local
   [Fill in Supabase keys]

2. DATABASE (5 min)
   - Create Supabase project (https://supabase.com)
   - Run SUPABASE_SCHEMA.sql in SQL Editor
   - Create song-files bucket in Storage

3. RUN LOCALLY (2 min)
   npm run dev
   Open http://localhost:3000

4. DEPLOY (5 min)
   - Push to GitHub
   - Import in Vercel (https://vercel.com)
   - Set environment variables
   - Deploy

                           KEY FEATURES
================================================================================

AUTHENTICATION:
- Shared password gate (hardcoded, no accounts)
- Session cookies (7-day expiry)
- Identity selection (Coris or Al)

SONG MANAGEMENT:
- Create songs with titles
- Upload multiple MP3 versions
- Rename songs and versions
- Version auto-numbering

WAVEFORM PLAYER:
- WaveSurfer.js integration
- Play/pause controls
- Click-to-comment on waveform
- Real-time playback seeking
- Waveform visualization

COMMENTS:
- Timestamped comment threads
- Click waveform to create at specific time
- Thread replies (threaded conversations)
- Comments version-specific
- No editing/deletion (v1)

NOTIFICATIONS:
- Email notifications via Resend
- Sent only to other person
- Includes song name, version, timestamp
- Deep link to thread in email
- HTML and text formats

DEEP LINKING:
- Share exact thread links
- Auto-login before redirect
- Automatic thread opening
- Playback seeks to timestamp

UI/UX:
- Minimal, clean design
- Black and white only
- Helvetica font
- Admin tool aesthetic
- No gradients, shadows, or animations
- Responsive layout
